Methodology

How Sentinel Works

A four-step pipeline from URL to plain-language explanation.

01

Feature Extraction

16 structural features are automatically extracted from your URL — no page load required. Includes IP detection, HTTPS status, subdomain depth, TLD reputation, and more.

02

ML Prediction

A Random Forest classifier (200 trees, 95.5% accuracy) trained on 11,000+ real phishing and legitimate websites generates a prediction with a calibrated probability score.

03

SHAP Explainability

SHapley Additive exPlanations (SHAP) calculate the exact contribution of each feature to the prediction. TreeExplainer provides provably exact values — not approximations.

04

Plain-Language Report

Technical SHAP values are translated into human-readable explanations. Any user — not just security experts — can understand exactly why a URL was flagged.

Explainable AI

Transparency over black boxes

Standard machine learning models are opaque — they produce a verdict without justification. In cybersecurity, trust requires explanation.

Sentinel uses SHAP (SHapley Additive exPlanations), a game-theory-based method that assigns each feature a contribution score. You can see exactly which URL properties pushed the model toward "phishing" and which pushed toward "legitimate."

This is not just academically interesting — it lets security analysts verify decisions, and lets ordinary users understand why a link is unsafe.
