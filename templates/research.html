Why 16 Features Instead of 30?
Prediction Integrity

The original 30-feature model requires full page crawling to measure iframes, popup behavior, right-click status, and anchor ratios. Substituting these with neutral defaults (0) feeds the model out-of-distribution inputs it was never trained to handle — degrading confidence scores and making SHAP values misleading.

SHAP Honesty

When using Option B (16 features), every SHAP value shown in the explainability dashboard corresponds to a feature that was actually measured. An examiner pointing at any bar in the waterfall chart can be given a complete, honest answer about how the value was obtained.

Industry Precedent

Production phishing detectors (including Google Safe Browsing's first-pass filter) use URL-structural features as the primary signal precisely because they are fast, reliable, and require no page load. Our approach is consistent with real-world engineering practice.

Accuracy Preserved

URL-structural features are the strongest predictors in the UCI dataset. The deployed 16-feature model achieves ~95.5% accuracy — within 1.3% of the paper's full 30-feature Random Forest result — while being faster, more reliable, and fully deployable without a browser or crawling infrastructure.

SHAP Explainability Methodology

What is SHAP?

SHAP (SHapley Additive exPlanations) is a game-theoretic approach to explaining machine learning model outputs. Introduced by Lundberg & Lee (2017), it assigns each feature a contribution value for a specific prediction.

Unlike simpler methods, SHAP satisfies three important properties: local accuracy (contributions sum to the prediction), missingness (missing features have no contribution), and consistency (more impactful features always get higher values).

TreeExplainer

Sentinel uses shap.TreeExplainer — optimized for tree-based models like Random Forest. Unlike sampling-based SHAP variants, TreeExplainer computes exact SHAP values in polynomial time by leveraging the tree structure.

This means every SHAP value displayed is mathematically exact — not an approximation. For a 16-feature Random Forest with 200 trees, this runs in milliseconds.
